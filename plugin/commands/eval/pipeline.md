# /eval:pipeline

Run integrity checks on data pipelines.

## Usage
- `/eval:pipeline all` — check all configured pipelines
- `/eval:pipeline product-catalog` — check a specific pipeline
- `/eval:pipeline orchestration-dashboard` — check a specific pipeline
- `/eval:pipeline --snapshot product-catalog` — save current outputs as golden files

## Instructions

1. Parse the argument.  If `--snapshot` is present, run the snapshot flow (step 6).  Otherwise continue to step 2.

2. Load config from `~/.ai-evals/config.yaml` (falls back to defaults if absent).

3. Run the pipeline checks using the framework's Python module:

```bash
cd /Users/ahmedm/Developer/ai-evals-framework
python3 -c "
from core.pipeline_checks import run_all_pipelines, format_pipeline_report
results = run_all_pipelines(pipeline_names=PIPELINE_ARG)
print(format_pipeline_report(results))
"
```

Replace `PIPELINE_ARG` with:
- `None` when the argument is `all` (checks every configured pipeline)
- `['product-catalog']` (or whichever name was given) for a single pipeline

4. Display results as a markdown table with traffic-light indicators.  Use this layout:

```
| Pipeline | Check | Status | Value | Threshold |
|----------|-------|--------|-------|-----------|
| product-catalog | schema_valid | OK   | dict  | -         |
| product-catalog | freshness    | WARN | 52h   | <48h      |
```

5. If any CRITICAL checks fail, highlight them prominently at the top of the output:

```
## Critical Failures
- [product-catalog] file_present: MISSING: moments.json
```

6. **Snapshot flow** (`--snapshot`):

```bash
cd /Users/ahmedm/Developer/ai-evals-framework
python3 -c "
from core.golden import snapshot_pipeline_golden
saved = snapshot_pipeline_golden('PIPELINE_NAME')
for s in saved:
    print(f'  Saved: {s}')
"
```

7. Append the raw check results (as JSONL) to `~/.ai-evals/results/YYYY-MM-DD.jsonl` so the aggregator can pick them up.

## Output Format

The full report is generated by `format_pipeline_report()` and includes:
- Summary counts (total, passed, critical, warnings)
- Per-pipeline detail table
- Critical failure callout section
- Timestamp

## Configuration

Pipeline configs are stored in `core/pipeline_checks.py` as defaults and can be overridden in `~/.ai-evals/config.yaml` under the `pipelines:` key:

```yaml
pipelines:
  product-catalog:
    freshness_hours: 72
    row_count_golden:
      moments.json: 20
      push-performance.json: 50
  my-custom-pipeline:
    name: My Pipeline
    data_dir: ~/data/output/
    expected_files: ["report.json"]
    freshness_hours: 12
```
